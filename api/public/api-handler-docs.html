IDP API Handler – File Upload APIs

Use these endpoints to upload and fetch files for an API channel Process Instance. The API validates your token and restricts access to the Process Instance folder.

This page is served from the backend public folder. If your backend base URL is http://localhost:3000, open: /api-handler-docs.html

Base URL

Replace {{BASE_URL}} in examples with your backend URL (example: http://localhost:3000). 

{{BASE_URL}} = http://localhost:3000

Authentication

Pass the Process Instance token in one of the following ways:

  • Recommended: Authorization: Bearer <token>
  • Or header: token: <token>
  • Or query: ?token=<token>
Token is generated when you create an API-channel Process Instance (short-lived), and you can optionally generate a long-lived token (60 days) using the endpoint below.

1) Verify Token

GET /api-handler/verify-token 
curl -s "{{BASE_URL}}/api-handler/verify-token" \
  -H "Authorization: Bearer <token>"
Returns: { valid, processInstanceId, processInstanceName, processId, folderName }

2) Generate Long-lived Token

POST /api-handler/generate-lived-token 
curl -s -X POST "{{BASE_URL}}/api-handler/generate-lived-token" \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d "{}"
Returns: { longLivedToken, folderName }

3) Upload Files

Upload files to the Process Instance folder using multipart/form-data. Folder access is restricted to the folder bound to your token.

POST /api-handler/files/{folderName} 
curl -s -X POST "{{BASE_URL}}/api-handler/files/{folderName}" \
  -H "Authorization: Bearer <token>" \
  -F "files=@C:\path\to\document1.pdf" \
  -F "files=@C:\path\to\image1.png"
Returns: uploaded file metadata + URLs.

4) List Files

List my folder

GET /api-handler/files 
curl -s "{{BASE_URL}}/api-handler/files" \
  -H "Authorization: Bearer <token>"

List specific folder

GET /api-handler/files/{folderName} 
curl -s "{{BASE_URL}}/api-handler/files/{folderName}" \
  -H "Authorization: Bearer <token>"
Returns: [ "file1.ext", "file2.ext", ... ]

5) Download File

GET /api-handler/files/{folderName}/{fileName} 
curl -L "{{BASE_URL}}/api-handler/files/{folderName}/{fileName}" \
  -H "Authorization: Bearer <token>" \
  -o "{fileName}"

6) Webhook “Get Info”

Returns ready-to-use URL templates and headers to help integrate with HTTP/HTTPS webhook tools.

GET /api-handler/webhook/get-info 
curl -s "{{BASE_URL}}/api-handler/webhook/get-info" \
  -H "Authorization: Bearer <token>"
